# 实体链接与知识对齐智能体 - 定义与规划

---

## 一、智能体定义

### 1.1 智能体名称

**实体链接与知识对齐智能体**（Entity Linking and Knowledge Alignment Agent）

### 1.2 智能体定位

本智能体是数据治理流水线中的**专项能力智能体**（课题10），在归一环节承担核心实体对齐能力。

```
数据治理流水线：
解析 → 清洗 → 归一 → 转换组织 → 质量校验 → 持续迭代
                ↑
            本智能体
```

### 1.3 核心功能

将文本中的实体指称（mention）链接到知识库中的唯一标准实体，完成：
- **消歧**：区分同名异指（如不同语境下的"国网"指代不同主体）
- **标准化**：把别名/简称/曾用名映射到标准全称与唯一ID
- **NIL检测**：在知识库中不存在该实体时正确判定为"无对应"

### 1.4 输入输出

**输入**：
- 文本内容
- 实体指称（mention）：文本、位置、类型
- 知识库

**输出**：
- 链接结果：标准实体ID、标准名称、置信度
- 追溯信息：链接依据、处理过程

### 1.5 服务接口

对外暴露标准HTTP API接口，支持热插拔集成：

```
POST /link          - 单条实体链接
POST /batch_link    - 批量实体链接
GET  /trace/{id}    - 查询追溯日志
GET  /health        - 健康检查
GET  /kb/stats      - 知识库统计
```

## 三、技术方案

### 3.1 总体架构

采用五层分层架构：

```
┌─────────────────────────────────────────────────────────────┐
│                      API接口层                               │
│            FastAPI RESTful 接口                              │
└────────────────────────────┬────────────────────────────────┘
                             │
┌────────────────────────────▼────────────────────────────────┐
│                    编排调度层                                 │
│            Pipeline 流程控制                                 │
└────────────────────────────┬────────────────────────────────┘
                             │
┌────────────────────────────▼────────────────────────────────┐
│                    核心能力层                                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐       │
│  │ 候选检索  │ │ 上下文消歧│ │ NIL检测  │ │ 追溯记录  │       │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘       │
└────────────────────────────┬────────────────────────────────┘
                             │
┌────────────────────────────▼────────────────────────────────┐
│                    基础设施层                                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐                    │
│  │ 知识库    │ │ BM25索引  │ │ 别名词典  │                    │
│  └──────────┘ └──────────┘ └──────────┘                    │
└────────────────────────────┬────────────────────────────────┘
                             │
┌────────────────────────────▼────────────────────────────────┐
│                    数据模型层                                 │
│       Mention / Entity / Candidate / LinkResult             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 核心模块设计

#### 3.2.1 知识库管理器（KnowledgeBase）

**职责**：加载和管理知识库数据

**功能**：
- 加载知识库JSON文件
- 加载别名词典
- 构建名称索引、类型索引
- 提供实体查询接口

**接口**：
```python
class KnowledgeBase:
    def load(knowledge_base_path, aliases_path)
    def get_entity(entity_id) -> Entity
    def get_entity_by_name(name) -> Entity
    def search_by_alias(alias) -> List[Entity]
    def search_by_type(entity_type) -> List[Entity]
```

#### 3.2.2 BM25索引（BM25Index）

**职责**：提供全文检索能力

**功能**：
- 对知识库实体建立BM25倒排索引
- 支持中文分词
- 提供检索接口

**接口**：
```python
class BM25Index:
    def build(entities)
    def search(query, top_k) -> List[Tuple[entity_id, score]]
```

#### 3.2.3 消歧器（Disambiguator）

**职责**：从候选中选择最佳实体

**消歧信号**：
- 类型一致性
- 名称匹配度
- 关键词重叠度
- 实体流行度

**接口**：
```python
class Disambiguator:
    def disambiguate(mention, candidates, top_k) -> List[Candidate]
```

#### 3.2.4 NIL检测器（NILDetector）

**职责**：判断是否为NIL

**检测信号**：
- 候选分数是否过低
- 类型是否一致
- 候选分数方差是否过小

**接口**：
```python
class NILDetector:
    def detect(mention_text, mention_type, candidates, best_candidate) -> NILResult
```

#### 3.2.5 追溯日志（TraceLogger）

**职责**：记录处理过程

**记录内容**：
- 原值（mention文本）
- 新值（链接到的实体）
- 依据（匹配方式、分数）

**接口**：
```python
class TraceLogger:
    def create_trace(mention_id, mention_text, entity_type) -> TraceLog
    def save_trace(trace)
    def get_trace(trace_id) -> TraceLog
```

### 3.3 数据模型设计

#### 3.3.1 Mention（实体指称）

```python
@dataclass
class Mention:
    id: str                    # 唯一标识
    text: str                  # 指称文本
    start_pos: int             # 起始位置
    end_pos: int               # 结束位置
    entity_type: str           # 实体类型
    context: str               # 上下文
```

#### 3.3.2 Entity（知识库实体）

```python
@dataclass
class Entity:
    id: str                    # 唯一ID
    standard_name: str         # 标准全称
    entity_type: str           # 实体类型
    aliases: List[str]         # 别名列表
    description: str           # 描述
    popularity: float          # 流行度
```

#### 3.3.3 LinkResult（链接结果）

```python
@dataclass
class LinkResult:
    mention: Mention           # 原始指称
    linked_entity: Entity      # 链接实体
    is_nil: bool               # 是否NIL
    confidence: float          # 置信度
    nil_reason: str            # NIL理由
    trace_id: str              # 追溯ID
```

### 3.4 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| Web框架 | FastAPI | 异步、自动OpenAPI文档 |
| BM25 | rank-bm25 | 轻量级、易集成 |
| 分词 | jieba | 中文分词成熟 |
| 存储 | JSON + 文件 | 轻量级、可私有化 |

### 3.5 与通用方案的差异化

| 方面 | 通用方案 | 本系统 |
|------|---------|--------|
| 知识库 | 固定Wikidata | 可配置知识库 |
| 流程 | 端到端黑盒 | 可配置Pipeline |
| 追溯 | 无 | 完整追溯日志 |
| 部署 | 依赖外部服务 | 可私有化部署 |
| 成本 | 全程大模型 | 能用规则不用大模型 |

```
输入：text + mention + knowledge_base
    │
    ▼
┌─────────────────┐
│ 1. 标准名称匹配  │ ← 最高优先级
└────────┬────────┘
         │ 未命中
         ▼
┌─────────────────┐
│ 2. 别名精确匹配  │
└────────┬────────┘
         │ 未命中
         ▼
┌─────────────────┐
│ 3. 模糊匹配     │ ← 名称包含关系
└────────┬────────┘
         │ 未命中
         ▼
┌─────────────────┐
│ 4. BM25检索     │ ← 全文匹配
└────────┬────────┘
         │ 未命中
         ▼
┌─────────────────┐
│ 5. NIL判定      │
└────────┬────────┘
         │
         ▼

```

## 八、总结

本智能体已按照任务书要求完成核心功能的实现，包括：

1. **候选实体检索**：别名匹配 + BM25全文检索
2. **上下文消歧**：类型过滤 + 名称最短优先
3. **实体标准化**：别名→标准全称+唯一ID
4. **NIL检测**：多信号融合判定
5. **可追溯**：原值→新值→依据

当前准确率84.45%，主要错误来源是知识库中的全称/简称混淆。后续可通过合并冗余实体将准确率提升到94%+。
