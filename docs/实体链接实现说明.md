# 实体链接与知识对齐智能体 — 实现说明

## 一、系统概述

本系统将文本中的实体指称（mention）链接到知识库中的标准实体，完成消歧、别名标准化、NIL检测与共指消解。

**输入**：文本 + 已识别的实体指称（mention）+ 知识库（含标准实体、别名）  
**输出**：每个指称链接到的标准实体（标准全称 + 唯一 ID）；消歧与共指结果；NIL 判定；链接依据

---

## 二、实体链接流水线

### 2.1 总体流程

```
输入: mention_text + entity_type + 文章全文
         │
         ▼
第1步: 标准名称精确匹配
  → knowledge_base.get_entity_by_name(mention_text)
  → 命中则返回，置信度 1.0
         │ 未命中
         ▼
第2步: 别名精确匹配 + 消歧
  → knowledge_base.search_by_alias(mention_text)
  → 单候选 → 置信度 0.95，直接返回
  → 多候选 → 消歧器 5 信号评分 → 返回最佳
         │ 未命中
         ▼
第3步: 模糊匹配（名称包含关系）
  → 遍历 KB，找 mention ⊂ standard_name 或反之，长度差 ≤8
  → 单候选 → 置信度 0.80
  → 多候选 → 消歧器评分
         │ 未命中
         ▼
第4步: BM25 全文检索
  → jieba 分词 + rank-bm25 打分，top-10
  → 置信度 ≤0.70
         │ 未命中
         ▼
第5步: NIL 判定
  → 返回知识库中不存在该实体
```

### 2.2 核心代码

```
elagent/api/routes.py — _enhanced_link() 函数（5级瀑布）
elagent/core/knowledge_base.py — 知识库加载与检索
elagent/core/disambiguator.py — 消歧器（5信号）
elagent/core/bm25_index.py — BM25 索引
elagent/core/nil_detector.py — NIL 检测器
```

## 三、消歧器（5 信号加权评分）

### 3.1 权重

| 信号 | 权重 | 说明 |
|---|---|---|
| 名称匹配 | 0.15 | 精确/别名/包含关系匹配 |
| 先验概率 | 0.15 | cbrt 归一化，标注数据出现频率 |
| 名称完整度 | 0.09 | 全称优先，过度细化惩罚 |
| 类型一致性 | 0.15 | PER/ORG/LOC/EVENT 硬过滤 |
| 领域匹配 | 0.10 | 2+3gram 区分词在全文中的命中密度 |

### 3.2 名称匹配

```python
mention == standard_name:
  ≤2字 → 0.75（简称惩罚）
  3字  → 0.80
  4字  → 0.85
  5字  → 0.90
  >5字 → 1.00
mention 在别名中 → 0.95
标准名包含 mention → 0.70~0.95（越接近全称越高）
mention 包含标准名 → 0.60~0.70
```

短名称惩罚：当 mention 长度 > 标准名长度时，乘以 `len(standard_name)/len(mention_text)`（最低 0.5）

### 3.3 先验概率

```python
prob = prior_probs.get(entity_id, 0.0)
# cbrt 归一化：筛除高频频响，缩小中频差距
normalized = (prob / max_prob) ** (1/3)
# 示例：游泳 0.00215→0.523，羽毛球 0.00059→0.339
# 比值 1.54:1（log缩放是 2.5:1）
```

### 3.4 名称完整度

```python
标准名 > mention × 1.5 → 0.90（全称）
标准名 > mention × 1.2 → 0.80
标准名 ≥ mention      → 0.70
标准名字符是 mention 的子集 → 0.85（mention过度指定）
其他                → 0.30（实体是简称）
```

过度细化惩罚：标准名 > mention × 1.5 → domain_score 减半

### 3.5 领域匹配

```python
# 2+3gram 区分词在全文中的命中密度
entity_bigrams = {"浙江","江省","省羽","羽毛","毛球","球队","浙江羽","羽毛球队"}
mention_bigrams = {"浙江","江队","浙江队"}
diff = entity_bigrams - mention_bigrams  # {"省羽","羽毛","毛球","球队","羽毛球队"}
total_hits = sum(full_text.count(ng) for ng in diff)
density = total_hits / (len(full_text) / 100)
score = min(density / 5.0, 1.0)
```

### 3.6 总分

```python
total_score = 0.15×name_match + 0.15×prior + 0.09×completeness + 0.15×type + 0.10×domain
```

## 四、NIL 检测

### 4.1 触发时机

瀑布流第 5 步——前 4 步均未找到匹配时触发。

### 4.2 判定逻辑

```python
nil_detector.detect(mention_text, mention_type, candidates, best_candidate):
  信号1: 无候选 → NIL（置信度 0.95）
  信号2: 最佳分数 < 0.4 → NIL（置信度 0.8）
  信号3: 类型不一致 → NIL（置信度 0.7）
  信号4: 候选分数方差 < 0.1 → NIL（置信度 0.6）
```

当前实际只用到信号 1——前 4 步都没找到候选时，无候选→NIL。

### 4.3 评测方式

独立数据集 `Dataset/NIL.json`（104 篇赛车文章，2,527 条 NIL mention）。  
不构建 BM25 索引时准确率 83.18%，构建 BM25 时降为 30.43%（BM25 对赛车术语产生假阳性匹配）。

## 五、共指消解

### 5.1 实现

纯规则驱动，不依赖模型。

```python
# 步骤1：识别指代词
人称代词: "她"→PER, "他"→PER, "它"→ORG, "其"→PER
指示代词: "本次赛事"→EVENT, "该队"→ORG, "该地区"→LOC（共 29 种模式）

# 步骤2：回链
def resolve_coreference(mention_index, all_mentions):
    策略1: 同类型 + 非指代 mention（最近前序）
    策略2: 任意非指代 mention（兜底）
    策略3: 同类型指代（链式回指）
```

### 5.2 评测

独立评测 5,916 个指代词，准确率 94.3%。

## 九、设计原则

1. **弱自主规划**：主流程由模板固化（5 级瀑布），确保结果可复现
2. **强流程约束**：每个 Skill 有明确输入输出，状态机控制流转
3. **能用规则不用 LLM**：BM25、别名匹配、NIL 阈值均为纯规则
4. **可溯源可审计**：每一步留痕（原值→新值→依据），支持回放和回滚
