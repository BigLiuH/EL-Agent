## 一、项目概述

### 1.1 课题背景

在数据治理流水线中，实体链接是归一环节的核心能力。本课题聚焦于"识别之后"的链接、消歧与标准化，将文本中的实体指称链接到知识库中的唯一标准实体。

### 1.2 研究目标

设计并实现一个实体链接与知识对齐智能体：
- 在已给定文本与实体指称的前提下，把每个指称链接到知识库中的标准实体
- 完成消歧、别名/简称/曾用名标准化
- 实现NIL检测
- 保留链接依据以供追溯

## 三、核心功能实现

### 3.1 候选实体生成/检索

采用多路召回策略：

1. **标准名称匹配** - 最高优先级
2. **别名精确匹配** - 次优先级
3. **模糊匹配** - 名称包含关系
4. **BM25检索** - 全文匹配
5. **向量检索** - 语义相似度

### 3.2 上下文消歧

采用分层消歧策略：

1. **类型过滤** - 优先选择类型一致的实体
2. **名称长度** - 优先选择名称更完整的实体
3. **候选分数** - 选择分数最高的候选

### 3.3 NIL检测

采用多信号融合判定：

1. **是否有候选** - 无候选则为NIL
2. **最佳候选分数** - 分数过低则为NIL
3. **类型一致性** - 类型不一致则为NIL

### 3.4 可追溯

记录每次链接的完整处理过程：

```json
{
  "trace_id": "uuid",
  "mention_text": "宁夏",
  "steps": [
    {
      "step_name": "标准名称匹配",
      "original_value": "宁夏",
      "new_value": "宁夏 (LOC_0873)",
      "reason": "标准名称完全匹配"
    }
  ],
  "final_result": {
    "linked_entity_id": "LOC_0873",
    "is_nil": false,
    "confidence": 1.0
  }
}
```

## 六、难点与解决思路

### 6.1 全称/简称混淆

**难点**: 知识库中同一实体有多个名称变体

**解决思路**:
1. 合并冗余实体
2. 优先选择名称更完整的实体

### 6.2 同名异指

**难点**: 同一名称在不同语境下指代不同主体

**解决思路**:
1. 类型过滤
2. 上下文消歧（待实现）

### 6.3 NIL检测

**难点**: 如何正确识别知识库中不存在的实体

**解决思路**:
1. 多信号融合判定
2. 设置合理的阈值
